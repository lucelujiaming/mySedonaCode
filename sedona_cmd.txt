上界 <? extend Fruit> ，表示所有继承Fruit的子类。
但是具体是哪个子类，无法确定，所以调用add的时候，要add什么类型，谁也不知道。
但是get的时候，不管是什么子类，不管追溯多少辈，肯定有个父类是Fruit。
所以，我都可以用最大的父类Fruit接着，也就是把所有的子类向上转型为Fruit。

下界 <? super Apple>，表示Apple的所有父类，包括Fruit，一直可以追溯到老祖宗Object 。那么当我add的时候，我不能add Apple的父类，因为不能确定List里面存放的到底是哪个父类。但是我可以add Apple及其子类。因为不管我的子类是什么类型，它都可以向上转型为Apple及其所有的父类甚至转型为Object 。但是当我get的时候，Apple的父类这么多，我用什么接着呢，除了Object，其他的都接不住。
所以，归根结底可以用一句话表示，那就是编译器可以支持向上转型，但不支持向下转型。具体来讲，我可以把Apple对象赋值给Fruit的引用，但是如果把Fruit对象赋值给Apple的引用就必须得用cast。

// 这个算法的思想是这样的。就是我们在绘制一条直线的时候，
// 会使用ROUND进行判断，也就是判断斜率是否大于0.5。
// 我们首先假设dx > dy，斜率是否大于0.5就相当于是判断dx/dy是否大于0.5。
// 那么就相当于判断2 * dx / dy是否大于1。

/************************************************************************/
/* 这个算法的思想其实还是没有脱离这两个原则。即：                       */
/*     每一步。无论如何，我们都会沿递增方向每递增一个单位，             */
/*     这会导致误差累加2 * dy。                                         */
/*     当误差大于零的时候，我们沿递增方向的垂直方向递增一个单位，       */
/*     这会导致误差减去2 * dx。                                         */
/************************************************************************/
/* 中点算法综合这两种情况，同时做了反向。就是：                         */
/*     每一步。无论如何，我们都会沿递增方向每递增一个单位，             */
/*     如果上一轮的误差小于零，则本轮会同时减少2 * dy和增加2 * dx。     */
/*     如果上一轮的误差大于零，则本轮只会减少2 * dy。                   */

以这句为例：
    char * str1 = "AAA";
首先这句话包含两个符号。
	左值是一个是.data节分配存储空间的初始化全局变量。或者是一个定义在栈上的局部变量。
	右值是在.rdata节分配存储空间的字符串常量。
这句话的处理逻辑如下：
我们首先调用type_specifier读取char发现Type是一个T_CHAR。
之后我们调用declarator，发现了*，我们会把Type改为T_PTR。
之后我们读取左值。之后再读一个符号。这里存在一个技巧。就是：
	如果是等号。说明是赋值操作。需要再读一个符号，把右值也读进来。
	此时，当前的token是右值字符串"AAA"。
之后我们继续处理左值。调用allocate_storage分配空间。
	如果是全局变量。就分配在节上。如果是局部变量，就分配在栈上。
	因为是指针，因此上分配四个字节。
如果没有赋值操作，这里处理就已经结束了。

但是如果存在赋值操作。我们会调用initializer函数。这时，前面的操作就有意义了。
	1. 这个函数的主分支是else部分。首先调用assignment_expression进行表达式分析。
	   因为右值，也有可能是一个表达式。但是我们这里因为是字符串。
	   因此上，在会进入primary_expression中处理。走TK_CSTR分支。
	   整个操作分为三步：
	   1.1. 生成一个字符串指针符号。
	   1.2. 调用allocate_storage分配空间。在这个函数中：	
	    	首先我们调用type_size计算需要分配的存储空间，这里有一个技巧。就是：
	    		如果是字符串"AAA"，他的长度是不固定的。这种情况下返回-1。
	    	之后，allocate_storage发现type_size返回-1.就会使用前面读到的右值计算数据长度。
	    	之后我们就可以利用前面计算出来的长度分配空间了。
	   1.3. 调用var_sym_put把变量放入符号表。
	   1.4. 嵌套调用initializer完成变量初始化。
	        因为前面调用allocate_storage分配空间的时候，是分配的全局空间。
	        因此上，allocate_storage的返回值sec不是空。同时我们强制指定了T_ARRAY类型标志。
	        这导致我们进入initializer函数的if部分。把字符串memcpy到节的指定位置。
	2. 之后，我们调用init_variable完成赋值操作。
	   因为此时右值已经压栈，我们只需要把左值压栈。之后swap左值和右值。
	   之后生成机器码即可。

设一个人在张纸上画了一个长方形S，一半涂黑A, 一半涂白B。
“非黑即白（非A=B）” 这个结论， 对于生活在长方形S里的图形小人p0而言是直觉，是天经地义的。
“在这个长方形S作用域内，非黑即白才成立“， 这对于图形人而言就是经典逻辑，记为L1. 对于外部的纸片人p1或身处“更高维”的我们，这依旧是直觉，因为长方形S对于我们我们是直觉可知的。
逻辑的形成：直觉+对此直觉本身的抽象边界认知，p0小人只有不断对长方形S进行直觉探索，不断对这些区域进行非黑即白的判定，直到弄清自身原有认知宇宙（长方形S）不过是纸上一块区域，非黑即白仅在S之内才成立时，才会诞生逻辑。 因此题主的问题更透彻的疑问就是，为何西方的p0小人为何完成对直觉的补完，而我国古代没有。

方向错了。始基学的对象其实有很多种，比如万物本质是火、是水、是土等等，但不巧只有数才能引导人类一直探索下去，因为数是人类对于理性认知的具现：抽象的意义。中国古代最接近这个概念的是道生一、一生二、三生万物。 但遗憾的是并未更进一步独立出数始基的概念，更为流行的是更偏向直感的五行说等。为什么没有进一步呢? 可能是大环境不允许。
2 内陆中原地理环境中大一统对逻辑学发展的压制。
逻辑学发展，在生产力极度缺乏的古代，是一个极其奢侈的事情。对社会体现不了立竿见影的突破，需要极长一段新文明的孕育。 中国历史上大规模集中对直觉探索讨论有三次 春秋战国之际的百家争鸣、魏晋三国时期的清谈玄谈、两宋之际的理学发展。并非是中国圣人喜欢拍脑袋凭直觉，而是把大量时间耗费在讨论辨析直觉的名家 无一例外的被现实的经验斗争（暴力）直接淘汰，甚至后秦亦如是。（看一看商君书就明白了）。逻辑的作用是解决现有问题，发现更高层次问题。大一统需求是尽可能排除一切问题，因此解决办法要求既无逻辑、也无直觉，而是单纯的服从： 在确定的命题里，按照确定的规则执行确定的事。简单地说，中国的p0小人不探索了，干脆画了一个圈，规定子子孙孙就在圈里跑，就不会产生未知的问题。所有的精力都要用来通过暴力解决统一与维系。不仅排斥的是逻辑，甚至排斥S外的直觉，所有直觉、灵感必须落于这个S内。 观中国的古代历史，上述三次萌芽，与其说是萌芽不如说是人性被极度压抑后的天然释放，不过力度是一次比一次弱。清谈玄谈误国，这是很多人信以为真的真理，但根本不知道对于对错边界的补完才能形成逻辑，思考的出发点不同，结论不同。这种对边界的探索，是东方这片土地所不能容忍的。所以我观两宋理学都有一种看纸片人的绝望感：我知道他们也很想获悉宇宙的奥秘，但思维早已被儒教桎梏降维，其认知世界的工具都已腐朽。 格物致知，如果“格”本身就有问题？会结出什么样的果实？ 实不相瞒，如今我观后秦也有这种感觉。

3 儒家文化规则下的腐朽认知工具: 象形表意文字系统古汉语。
我曾写过一篇长文论述文字的本质：直感与逻辑的载体。简单的讲，文字的信息层级决定了使用者起点是图形小人P0，还是纸片人P1，决定了初始认知的大环境。很多人对此没有发觉的原因是各位生活在现代，用着现代汉语+字母语言+符号公式的混合文字系统，已经能够覆盖绝大部分逻辑的表达。 用纯古汉语试试？文字描述，演绎一下各种“量变如何质变”，描述下微分？这是没有可能的。 
因为象形表意文字系统中每一个字都有最基本的表意M{}， 而逻辑本质是边界，要的就是区分是M1{} 和M2{} 有何不同，如果缺乏逻辑工具来对构成M的{}进行概念映射，是没法深入研究的。 如果有一个新的概念X诞生，对于古汉语则是在[M]系统中寻找给人感觉差不多的字进行重复映射。 其结果就是表意的“噪音污染。比如 Radio， 现代汉语翻译为收音机，如果在古汉语构词法，就叫音，最多叫音机。此类建立于表意是非常难以清晰概括Radio 作为利用无线电传讯接收系统这一本质。其结果就是望文生义，你只需要认识“收”、“音”、“机” 三个汉字就能形成一个关于Radio的意义映射，而无需深入了解Radio的严肃远离和定义。 
更进一步的，在语法系统结构上的混乱： {M1 M2 M3 M4} 和 {M1 M4 M2 M3} 区不大别，都看得懂。 
为了讲述这一概念，我都不得不现代汉语+字母语言+符号进行阐释，而在古代呢？禁止自由创造文字的古代呢？整个知识系统都遵循复古的古代呢？，没有一丝可能。 （BTW 2023年 国考居然还有一道题还是号召反对网络造词的） 

可以说自此分道扬镳：
西方逻辑学L0->L1->L2 不断在完成边界的收束后的破壁。 
东方知识分子，依然在画地为牢的圆圈中循环。
